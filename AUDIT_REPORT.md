## 1. Файловая структура

```

│   ├── fake_serial.py      # Эмулятор COM-порта
│   └── update_checker.py   # Проверка обновлений через GitHub API
├── models/

    ├── can_monitor_tab.py  # Страница «Мониторинг»
    ├── can_trigger_tab.py  # Страница «Триггеры» (таблица + диалог)

    └── library_tab.py      # ⚠️ больше не используется
```

## 2. Анализ интерфейса (`ui/`)

### 2.1. Главное окно (`main_window.py`)

**Визуальное описание:**
- Верхняя панель высотой 48 px, тёмно-синего цвета (`#252538`), слева — логотип «🛠️ Код Мастер», справа — индикатор порта, кнопки «⚙️ Настройки», «📄 Логи», переключатель темы ☀/☾, язык EN/RU, обновления 🔄 и «✕ Выход».
- При запуске показывается **мастер подключения** (`StartupWidget`) на весь центр окна: заголовок «Код Мастер», подзаголовок «Настройка подключения», информация о порте и две большие кнопки «Обновить» / «Настроить».
- После успешного подключения мастер скрывается и появляется **рабочий режим**:
  - Горизонтальная панель с 3 карточками-вкладками: «⚡ Триггеры», «🔍 Мониторинг», «🧩 Гибкая логика».
  - Под ней — `QStackedWidget` с соответствующими страницами.
  - Внизу — статус-бар с текстом состояния и версией.

**Статус:** ✅ Реализовано. Логика переключения режимов через два вложенных `QStackedWidget` корректна.

### 2.2. Страница «Мониторинг» (`can_monitor_tab.py`)

**Визуальное описание:**
- Верхний ряд кнопок: «Запустить оба», «Остановить оба», «Очистить всё», «Записать в CSV».
- Панель отправки/триггера: поля «ID», «Данные», спин-боксы «Повторы» (1–100) и «Интервал, мс» (1–1000), кнопка «Отправить», чекбокс «Триггер», поле «Маска HEX».
- Две панели каналов CAN1/CAN2 в `QSplitter`:
  - Кнопки «Старт/Стоп/Очистить», фильтр ID, чекбокс «Приостановить».
  - Таблица 11 столбцов: Время, ID, D0…D7, DLC.
  - Панель ручной отправки с 8 отдельными полями D0…D7.
  - Статистика «Принято: N | Скорость: N пак/с».
- Контекстное меню таблицы: «Копировать ID», «Копировать данные», «Создать триггер».

**Функционал:**
- ✅ Отображение входящих CAN-кадров.
- ✅ Фильтрация по ID, пауза, очистка.
- ✅ CSV-запись.
- ✅ Глобальная отправка с повторами и интервалом.
- ✅ Режим «Триггер»: автоответ на входящий пакет с заданным ID; поддержка маски данных.
- ⚠️ При одновременной работе глобальной панели и пер-канальной панели отправки возможна путаница у пользователя.

### 2.3. Страница «Триггеры» (`can_trigger_tab.py`)

**Визуальное описание:**
- Заголовок «Триггеры CAN».
- Таблица с колонками: №, Условие, Действие, Статус, Срабатываний.
- Кнопки «Добавить», «Редактировать», «Удалить», «Применить», «Остановить», «Сохранить в файл», «Загрузить из файла».
- Модальный диалог `TriggerEditDialog` с полями: активность, ID условия, данные условия, ID ответа, данные ответа, задержка.

**Функционал:**
- ✅ Табличное управление триггерами.
- ✅ Добавление/редактирование/удаление.
- ✅ Активация/деактивация обработки.
- ✅ Автоматический ответ с задержкой.
- ✅ Экспорт/импорт JSON.
- ✅ Создание триггера из пакета мониторинга.

### 2.4. Страница «Гибкая логика» (`flexible_logic_tab.py`)

**Визуальное описание:**

- Таблица № / Условие / Действие / Статус с тремя фиктивными строками.

### 2.5. Тема (`dark_theme.py`)

**Визуальное описание:**

## 7. Критические и заметные проблемы

### 🔴 Критические
1. **Повреждён `CHECKLIST.md`.** Нужно восстановить/переписать.
2. **Страница «Прошивка» удалена из UI**, хотя бутлоадер реализован. Если прошивка нужна, требуется вернуть страницу или добавить пункт меню.
3. **Мёртвый код:** файлы `com_settings_dialog.py`, `settings_window.py`, `can_gateway_tab.py`, `firmware_tab.py`, `library_tab.py` не используются и могут вводить в заблуждение.

### 🟡 Заметные
4. **Страница «Гибкая логика» — только заглушка.** Нужна либо реализация, либо чёткая дорожная карта.
5. **Дублирование парсера CAN-кадров:** `serial_manager.py` не использует `parse_all_frames()` из `can_protocol.py`.
6. **Локализация:** многие новые строки в `can_monitor_tab.py`, `can_trigger_tab.py`, `flexible_logic_tab.py` и `main_window.py` (кроме вызовов `tr()`) захардкожены на русском.
7. **UX мастера подключения:** кнопка «Настроить» → «Сохранить» не всегда даёт понять, успешно ли подключение.
